# 双仓分区仓库出入库管理系统

## 项目简介

本项目是一套完整的双仓分区仓库出入库管理系统，适用于高校软件工程期末实训。系统实现了大仓（大批量、低周转、储备囤货）与小仓（小批量、高周转、零散领用、退换货）的联动管理，包含完整的申请审核流程、库存管理、数据溯源等功能。

## 技术栈

- 后端：Python + Flask + SQLite
- 前端：HTML + CSS + JavaScript
- 登录认证：Flask-Login

## 核心功能

### 1. 大仓采购入库

大仓管理员可创建采购入库单，审核通过后自动增加大仓库存

### 2. 小仓申请管理

- 要货申请：小仓向大仓申请要货
- 退库申请：小仓滞销/临期商品退大仓
- 入库申请：小仓独立入库（客户退货、门店退仓）

### 3. 小仓出库管理

小仓操作员创建出库单，强制校验库存，审核通过后自动扣减库存

### 4. 审核流程

所有单据必须审核后才会变更库存，支持审核时修改数量、驳回并填写原因

### 5. 库存查询

- 大仓管理员：查看全仓库存
- 小仓操作员：仅查看本仓库存

### 6. 操作流水

完整记录所有操作，支持溯源

## 数据库说明

系统使用SQLite数据库，首次运行自动初始化：
- 仓库：1个大仓，2个小仓
- 商品：5种测试商品
- 用户：4个测试账号
- 库存：初始为0

## 业务规则

1. 库存变动仅在审核通过后生效
2. 大小仓数据物理隔离，权限精准控制
3. 申请审核通过自动生成对应出入库单
4. 出库前强制校验库存，防止超发
